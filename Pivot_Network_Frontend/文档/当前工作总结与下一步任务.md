## 2. 项目一句话定义

> **个人算力交易平台**：卖家将闲置算力接入平台并商品化为可售资源，买家选购算力后获得可控运行时环境以执行任务（训练、推理、遥感分析等），平台统一负责交易授权、会话编排与状态监控。

核心业务主链：

```
卖家接入 → 节点验收 → Offer商品化 → 买家下单 → 授权签发 → RuntimeSession创建 → Shell/Workspace/Task使用
```

### 3.3 第二阶段：静态前端 Demo（[pivot-demo-frontend/](../pivot-demo-frontend/)）

**技术栈：** 纯 HTML + CSS + JavaScript（零依赖，浏览器直接打开）

**已实现的 6 个页面模块：**

| 页面 | 功能 | 状态 |
|---|---|---|
| 平台总览 | 核心指标看板、资源流转说明、近期动态 | 完成 |
| 资源市场 | 3 个模拟 Offer 卡片、规格展示、下单按钮 | 完成 |
| 算法工坊 | 3 个算法模板（YOLOv11 / LoRA微调 / 遥感分割）、推荐资源匹配 | 完成 |
| 监控中心 | 4 个节点状态表、GPU/CPU 进度条、GPU 使用率趋势柱状图 | 完成 |
| 卖家控制台 | 资源发布表单（7 个字段）、已发布资源记录 | 完成 |
| 会话中心 | RuntimeSession 状态、授权码、任务列表、工作区、日志 | 完成 |

**买卖闭环交互流程：**
1. 在资源市场点击"立即开通" → 弹出下单抽屉
2. 选择算法模板 + 使用时长 → 点击"确认下单"
3. 自动生成 RuntimeSession → 页面跳转到会话中心
4. 会话中心展示会话状态、任务列表、工作区和日志

**存在问题（师兄反馈）：**
- 全部代码集中在 3 个文件中（`index.html` 699 行 JS、`styles.css` 679 行、`index.html` 133 行 HTML），零模块化
- 数据全部硬编码在 `initialState` 对象中，无数据层分离
- 页面切换通过字符串模板替换实现，无路由机制
- 无类型检查、无状态管理、无组件复用
- **结论：** 只能作为需求演示原型，无法作为正式开发基础

### 3.4 第三阶段：Vue3 工程化重构（[pivot-vue-frontend/](../pivot-vue-frontend/)）

**技术选型：**

| 技术 | 版本 | 用途 |
|---|---|---|
| Vue 3 | ^3.5.22 | 前端框架 |
| Vite | ^7.1.7 | 构建工具 |
| TypeScript | ~5.9.3 | 类型系统 |
| Pinia | ^3.0.3 | 状态管理 |
| Vue Router | ^4.5.1 | 路由管理 |

**工程结构（13 个源文件）：**

```
src/
├── main.ts                          # 应用入口
├── App.vue                          # 根组件（布局 + 侧边栏 + 下单抽屉）
├── router/index.ts                  # 路由配置（6 条路由，懒加载）
├── types/domain.ts                  # 8 个核心 TypeScript 接口定义
├── stores/platform.ts               # Pinia Store（状态 + 7 个业务方法）
├── services/mock/platform.mock.ts   # Mock 数据层（6 类模拟数据）
├── assets/styles/main.css           # 全局样式（705 行）
└── views/                           # 6 个页面视图组件
    ├── OverviewView.vue
    ├── MarketplaceView.vue
    ├── AlgorithmsView.vue
    ├── MonitorView.vue
    ├── SellerConsoleView.vue
    └── SessionCenterView.vue
```

**相比 Demo 的改进：**

| 维度 | Demo 版本 | Vue3 版本 |
|---|---|---|
| 代码组织 | 3 个文件，无模块化 | 13 个文件，路由/视图/Store/Mock 分层 |
| 路由 | 字符串模板切换，无 URL | Vue Router，6 条独立路由，懒加载 |
| 状态管理 | 全局可变对象 `state` | Pinia Store，响应式 + computed |
| 类型安全 | 无 | 8 个 TypeScript 接口，覆盖全部业务对象 |
| 数据层 | 硬编码在 JS 中 | 独立 `services/mock/` 目录，可替换为 API |
| 样式 | 679 行单一 CSS | 705 行，增加细节打磨（hover 效果、响应式断点优化等） |
| 可迭代性 | 无法持续开发 | 可直接在现有结构上扩展组件和接口 |

**尚未完成的部分：**
- 组件粒度仍偏粗（页面视图未拆分为可复用子组件）
- 仅有一个 Store，未按领域拆分（offers / session / monitor）
- 未抽取 Composables（如 `useCheckout`、`useSessionActions`）
- 无真实 API 对接层（`services/api/` 目录为空）
- 无登录/认证流程
- 无加载态、空态、错误态处理
- 缺少单元测试

## 4. 对当前内容的综合评价

### 4.1 做得好的地方

1. **需求理解到位。** 从后端代码和文档中准确提取了业务模型和核心闭环，Demo 和 Vue3 版本都正确覆盖了买卖闭环、算法选择、算力监控三大要求。
2. **技术选型合理。** Vue3 + Vite + Pinia + TypeScript 是当前前端主流方案，适合持续迭代。
3. **工程化意识强。** 虽然 Demo 是纯静态，但 Vue3 版本已经完成了路由拆分、Store 分离、Mock 分离、类型定义，具备了正式项目的基本骨架。
4. **产品形态完整。** 6 个页面覆盖了买家、卖家、平台管理三类用户视角，交互闭环可完整演示。

### 4.2 需要改进的地方

1. **组件拆分不够。** 每个 View 仍是一个大文件（如 SessionCenterView 近 90 行模板），缺少可复用的卡片、表单、图表组件。
2. **Store 未按领域拆分。** 所有业务状态混在一个 `platform.ts` 中，后续接入真实接口后会变得难以维护。
3. **Mock 数据与真实接口之间还差一层 Adapter。** 目前 Store 直接引用 Mock 数据，缺少一个可配置的 Service 层来做 mock/real 切换。
4. **缺少边界状态处理。** 所有页面默认数据完美，没有 loading skeleton、empty state、error state、网络超时等处理。
5. **交互细节可打磨。** 如下单后 Toast 提示、表单校验、按钮防抖、页面过渡动画等细节可以增强产品感。
