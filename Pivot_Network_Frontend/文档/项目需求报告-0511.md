# 项目需求报告

更新时间：`2026-05-11`

## 1. 报告目的

用于明确当前项目的业务定位、建设目标、需求范围、已有工作基础与实施方向。

## 2. 项目结论

结合当前目录材料、已有需求整理与现有代码结构，项目的核心建设目标可以归纳为：

> 建设一套面向个人算力交易场景的前后端平台，支持算力资源接入、资源商品化、交易授权、运行时访问与平台管理，形成从卖家供给到买家使用的完整闭环。

当前目录的定位可明确如下：

- 根据`Pivot_Network_Backend-main/`：当前主线项目，承载算力交易、授权、运行时会话与安全接入主链。

目前需求分析重点围绕 `Pivot_Network_Backend-main` 所代表的“个人算力交易平台”展开；

## 3. 项目背景与建设必要性

当前项目试图解决以下几类实际问题：

- 算力提供者拥有闲置算力，但缺少统一接入、验收和售卖机制。
- 算力消费者希望像使用云主机一样，按需购买并进入可用运行环境。
- 平台需要对资源供给、订单授权、运行时访问和安全隔离进行统一控制。
- 平台需要具备基础的账户、权限、状态追踪和运维观测能力。
- 现有组网方案存在局限，尤其 `WireGuard` 在 Windows 端稳定性不足，后续需向自研组网或可替换组网控制层演进。

因此，该项目不是单一页面开发任务，而是一套同时涉及前端、后端、客户端、运行时控制与安全接入的综合平台建设任务。

## 4. 项目总体定位

### 4.1 总体定位

这是一个“个人算力交易平台”的 MVP 到正式化过渡项目。

### 4.2 核心业务价值

- 对卖家：把本地机器接入平台并转化为可售资源。
- 对买家：购买资源后获得一个可控、可隔离、可使用的运行时环境。
- 对平台：建立资源接入、交易授权、运行时管理和安全接入的统一控制体系。

### 4.3 最终系统形态

项目最终应形成以下三类能力：

- 卖家接入能力：支持算力节点加入平台并通过验收。
- 买家消费能力：支持资源浏览、下单、授权和运行时使用。
- 平台管理能力：支持状态查询、权限管理、异常排查和基础运营管理。

## 5. 项目角色与职责

### 5.1 算力提供者 Seller

- 登录平台
- 发起节点接入
- 完成本地环境准备与状态上报
- 提供待验收算力节点

### 5.2 算力消费者 Buyer

- 登录平台
- 浏览资源
- 创建订单并获取授权
- 建立运行时会话并执行任务

### 5.3 平台管理员

- 管理账户与权限
- 查看节点、订单、授权、会话状态
- 参与验收、纠偏与异常排查

### 5.4 平台后端与控制面

- 维护业务真相
- 编排接入、商品化、授权与会话流程
- 管理运行时与安全接入链路

## 6. 核心业务闭环

### 6.1 卖家接入闭环

1. 卖家登录本地客户端。
2. 平台创建接入会话。
3. 卖家获取接入材料并完成环境准备。
4. 节点加入平台并提交探测与完成状态。
5. 平台完成验收与能力评估。
6. 节点进入可售资源池。

### 6.2 资源交易闭环

1. 买家浏览资源列表。
2. 买家查看资源详情并创建订单。
3. 平台激活订单并生成访问授权。
4. 买家绑定授权并创建运行时会话。
5. 平台分配连接材料并建立受控访问链路。
6. 买家进入 shell、同步工作区并执行任务。

### 6.3 平台管理闭环

1. 平台读取节点状态、资源状态、订单状态与会话状态。
2. 平台支持错误摘要查看、会话排查与状态纠偏。
3. 管理端为后续运营、运维和权限治理提供支撑。

## 8. 非功能需求

### 8.1 安全性

- 买家不能绕过平台直接访问卖家宿主机。
- 访问必须通过授权和平台控制的安全接入链路完成。
- 卖家、买家、管理员权限边界必须清晰。

### 8.2 可靠性

- 会话中断后应支持恢复。
- 网络抖动下应尽量保持有限可用。
- Windows 端接入链路需要具备可验证的稳定性基线。

### 8.3 可观测性

- 节点、资源、订单、授权、会话状态可追踪。
- 平台保留探测、验收、错误和会话历史。
- 前端可读取必要的错误摘要与状态信息。

### 8.4 可维护性

- 明确正式目录与历史目录边界。
- 统一核心对象命名与接口语义。
- 提供可复现文档与最小测试链路。

## 9. 当前技术边界与约束

### 9.1 现有代码基础

当前已有 `Pivot` 主链原型，项目不是从零开始，已有以下基础：

- 卖家/买家/平台后端三段式结构
- `Offer`、`Order`、`AccessGrant`、`RuntimeSession` 等核心对象
- 本地客户端与后端职责划分
- 初步的运行时访问链路

### 9.2 当前主要限制

- 现有代码存在 bug 和过渡实现，工程稳定性不足。
- 目录中混有原型材料、历史材料和待收敛模块。
- 环境依赖较重，完整复现成本较高。
- 当前组网链路对 `WireGuard` 依赖较深，不适合作为长期正式方案。

### 9.3 组网方向约束

本项目后续在组网能力上应遵循以下原则：

- 长期方案应以自研组网能力为目标。
- 当前若继续复用 `WireGuard`，仅可视为过渡实现。
- 业务层、会话层与组网实现层必须解耦。
- 平台需要统一控制连接材料、路由、状态同步与主动关闭能力。

## 10. 核心接口需求

本节用于补充本项目最关键的接口输入输出要求。此处描述的是接口需求和交互规格，用于支撑前后端协作与后续实现。

### 10.1 认证接口

#### 10.1.1 用户注册

- 接口：`POST /auth/register`
- 调用方：Seller Client / Buyer Client / Web 前端
- 接口用途：创建平台用户并返回登录态

输入参数：

- `email`：用户邮箱
- `display_name`：显示名称
- `password`：登录密码
- `role`：用户角色，默认可为 `buyer`，卖家可为 `seller`

输出结果：

- `access_token`：访问令牌
- `token_type`：令牌类型，默认 `bearer`
- `expires_at`：过期时间
- `user`：当前用户信息，包含 `id`、`email`、`display_name`、`role`、`status`、`created_at`、`updated_at`

常见失败：

- `409`：邮箱已存在
- `422`：输入字段不合法

#### 10.1.2 用户登录

- 接口：`POST /auth/login`
- 调用方：Seller Client / Buyer Client / Web 前端
- 接口用途：用户登录并换取访问令牌

输入参数：

- `email`：用户邮箱
- `password`：登录密码

输出结果：

- `access_token`
- `token_type`
- `expires_at`
- `user`

常见失败：

- `401`：邮箱或密码错误

### 10.2 卖家接入接口

#### 10.2.1 创建接入会话

- 接口：`POST /seller/onboarding/sessions`
- 调用方：Seller Client
- 接口用途：创建卖家节点接入会话并获取加入材料

输入参数：

- `requested_offer_tier`：期望售卖规格，可选
- `requested_accelerator`：期望加速器类型，如 `gpu`
- `requested_compute_node_id`：期望节点标识，可选
- `expected_wireguard_ip`：期望组网 IP，可选

输出结果：

- `session_id`：接入会话 ID
- `status`：会话状态，如 `issued`、`probing`、`joined`、`verified`
- `one_time_token`：一次性接入凭证
- `swarm_join_material`：加入材料，包含 `join_token`、`manager_addr`、`manager_port`、`registry_host`、`registry_port`、`swarm_join_command`、`recommended_compute_node_id`、`expected_wireguard_ip`、`recommended_labels`、`next_step`
- `required_labels`：平台要求节点附带的标签
- `manager_acceptance`：管理侧验收状态
- `expires_at`、`created_at`、`updated_at`

常见失败：

- `403`：非卖家或管理员无权限
- `5xx`：底层适配器返回错误

#### 10.2.2 获取接入会话详情

- 接口：`GET /seller/onboarding/sessions/{session_id}`
- 调用方：Seller Client / 平台管理端
- 接口用途：查询接入进度、探测结果和验收状态

路径参数：

- `session_id`：接入会话 ID

输出结果：

- 返回完整 `JoinSessionRead`
- 重点字段包括 `probe_summary`、`container_runtime_probe`、`last_join_complete`、`manager_acceptance`、`minimum_tcp_validation`、`effective_target_addr`

常见失败：

- `404`：接入会话不存在
- `403`：无权限

#### 10.2.3 提交主机探测

- 接口：`POST /seller/onboarding/sessions/{session_id}/linux-host-probe`
- 调用方：Seller Client
- 接口用途：上报卖家主机基础环境探测结果

输入参数：

- `reported_phase`：上报阶段，如 `detect`、`prepare`
- `host_name`：主机名
- `os_name`：操作系统名称
- `distribution_name`：发行版名称
- `kernel_release`：内核版本
- `virtualization_available`：是否支持虚拟化
- `sudo_available`：是否具备 sudo 能力
- `observed_ips`：观测到的 IP 列表
- `notes`：补充说明
- `raw_payload`：原始探测结果

输出结果：

- 返回更新后的 `JoinSessionRead`
- 平台据此刷新 `probe_summary`

#### 10.2.4 提交底层环境探测

- 接口：`POST /seller/onboarding/sessions/{session_id}/linux-substrate-probe`
- 调用方：Seller Client
- 接口用途：上报 Docker、WireGuard、GPU、CPU、内存、磁盘等基础能力

输入参数：

- `docker_available`
- `docker_version`
- `wireguard_available`
- `gpu_available`
- `cpu_cores`
- `memory_gb`
- `disk_free_gb`
- `observed_wireguard_ip`
- `observed_advertise_addr`
- `observed_data_path_addr`
- `observed_ips`
- `notes`
- `raw_payload`

输出结果：

- 返回更新后的 `JoinSessionRead`
- 平台据此生成资源能力摘要

#### 10.2.5 提交接入完成

- 接口：`POST /seller/onboarding/sessions/{session_id}/join-complete`
- 调用方：Seller Client
- 接口用途：提交节点加入完成信息，触发平台验收

输入参数：

- `reported_phase`
- `node_ref` 或 `compute_node_id`：二者至少提供一个
- `observed_wireguard_ip`
- `observed_advertise_addr`
- `observed_data_path_addr`
- `notes`
- `raw_payload`

输出结果：

- 返回更新后的 `JoinSessionRead`
- 重点包含 `manager_acceptance`、`status`、`last_join_complete`

常见失败：

- `409`：输入冲突或状态不允许
- `404`：接入会话不存在
